Overview

This page turns the original TPay integration notes into a clean developer-facing HTML reference. All requests are JSON POST requests to the listed API endpoints.

Base domain https://api.tpay8.com/API

Authentication & Signature

TPay uses an MD5 signature generated from request parameters sorted in ascending order, then appended with token={secretkey}.

Signature = MD5(field_1=value_1&field_2=value_2&field_x=value_x&token={secretkey})

Merchant ID is provided by TPay. Secret Key is generated from the back office.

Response Codes & Common Errors

General

  • { result = false, msg = "Bad IP" } — Contact support for IP whitelisting
  • { result = false, msg = "Signature Not Match" }
  • { result = false, msg = "Unknown Error" } — Contact support
  • { result = false, msg = "OnMaintenance" }
  • { result = false, msg = "User OnMaintenance" }

SetDeposit

  • { result = true, msg = "success submit" }
  • { result = false, msg = "Already Exists", res=0 } — transactionId already submitted
  • { result = false, msg = "Already Exists", res=1 } — existing transaction already approved
  • { result = false, msg = "Already Exists", res=2 } — existing transaction already rejected
  • { result = false, msg = "", res=13 } — on maintenance or bank not available

SetWithdrawal

  • { result = true, msg = "Success submit", res=10 }
  • { result = false, msg = "Balance Not Enough", res=11 }
  • { result = false, msg = "Already Exists", res=12 } — transactionId already submitted
